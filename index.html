Phase 2: Analysis Context

As per the Interaction Design Process (Figure 5.1), this phase takes the "What is wanted" from Requirements (Phase 1) and performs Task Analysis and Ethnography.

Task Analysis Focus

The Execution: Breaking down user actions into sequential physical steps to understand the gap between human intent and the computer system's response.

The Flow: Mapping exactly how physical keyboard interrupts and mouse clicks translate into mathematical canvas operations.

Ethnography Report

Observing real-world physical drawing reveals that fixed marks cause anxiety. By treating lines as fluid data arrays, our system removes the fear of permanent errors.

Task Analysis Breakdown

Sequential User Steps

  • B Target (x,y) → Press key → Array mapped
  • M Hover near → Press key → Click to lock
  • D Hover near → Press key → Vertex bridged
  • R Press key → Canvas and memory cleared
  • Q Press key → Application cycle terminated

Ethnographic Translations

  • Targeted erasure prevents 'area-of-effect' damage
  • Liquid adjustment replaces physical erasing needs
  • Design paradigm shifts to rapid execution
